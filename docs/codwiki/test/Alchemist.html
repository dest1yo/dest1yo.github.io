Alchemist

介绍​

Alchemist 是一个由Scobalula制作，用来处理CoD叠加(Additive)动画、混合动画(常用于手势、滑铲)以及IK的工具。

目前只有命令行版本，确实用起来很费劲，但是作者在之后会制作GUI版本。

由于现在只是捐助者测试版，所以暂不放出软件下载地址，仅提供教程。

首先明确一点，现在只支持SE系列格式文件，也就是SEModel和SEAnim作为模型和动画。

在Readme.md文件中我们可以看到其他指令的使用方法。

一些指令参数会有一定的默认值，比如说默认左右手都会开启IK，IK骨骼的默认名称是tag_ik_loc_le和tag_ik_loc_ri，以及默认手臂骨骼链都将以COD16的骨架命名。

准备工作​

首先我们可以在软件目录新建2个文件夹，一个是 Animations ，一个是 Skeleton。

我们先把武器的模型配件合并起来，把合并好的武器模型semodel放进 Skeleton文件夹，然后再把我们的手臂模型semodel放进去。

注意

每把枪的每种运动动画都有可能是不同的叠加方式，并不代表例子和动画命名一定通用。

比如说我想叠加一个冲刺动画，一般来说我们需要准备：

  • idle(待机)
  • sprint_offset(倾斜偏移)
  • sprint_loop(跑步时的晃动循环)

以L86这把枪的冲刺动画为例，我们需要准备动画：

  • "vm_lm_lima86_idle.seanim"
  • "vm_lm_lima86_sprint_offset_additive.seanim"
  • "vm_lm_lima86_sprint_loop.seanim"

然后把我们想叠加的动画都放进 Animations文件夹里。
(当然你放别的位置写明路径也可以，这样只是为了方便管理。)

使用​

注意

动画文件名要加路径，而且必须先叠加偏移(offset)再叠加晃动(loop)

我们新建一个.bat文件，在里面写如下规则参数：

提示

Alchemist.CLI --input-file "idle动画名" --output-file "输出的动画名" --skeleton-path 模型文件夹名 --additive-layer-file "sprint_offset(倾斜偏移)动画名" --additive-layer-file "sprint_loop(跑步时的晃动循环)动画名"

例子中实际的参数是：

提示

Alchemist.CLI --input-file "Animations\vm_lm_lima86_idle.seanim" --output-file "output.seanim" --skeleton-path Skeleton --additive-layer-file "Animations\vm_lm_lima86_sprint_offset_additive.seanim" --additive-layer-file "Animations\vm_lm_lima86_sprint_loop.seanim"

在最后加一个换行和pause可以防止软件运行完之后直接关闭。

这时我们启动刚才写好的批处理文件运行软件，如果指令和资源都没问题，会在软件目录生成一个test.semodel和output.seanim。

我们可以直接把他们导入到软件中查看效果。

其他参数​

比如我想叠加出一个战术冲刺的动画，由于很多枪在战术冲刺时左手并不会举着枪，而是在空中挥舞。我们可以添加一个参数关闭左手的IK效果:

  • --enable-left-hand-ik "False"

如果我们有握把配件，我们应该添加一个手部的叠加动画（一般会命名为pose_xxx），并且指定新的IK骨骼名作为目标点，添加参数：

  • --left-hand-pose-file "包括路径的动画文件名"
  • --left-iktarget-bone-name "握把上IK骨骼的名字"
